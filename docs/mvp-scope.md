# MVP 范围

## 目标

完成一个可线上验收的最小闭环系统：

1. 用户上传单个 PDF 简历。
2. 系统解析 PDF 文本。
3. 系统抽取关键信息。
4. 用户输入岗位 JD。
5. 系统返回匹配评分和关键词分析。
6. 前端页面可公开访问。

## 必做功能

### 模块一：简历上传与解析

- 支持上传单个 PDF 文件。
- 支持多页 PDF。
- 提取 PDF 文本。
- 清洗文本，包括空白字符、重复换行、不可见字符等。
- 返回原始文本、清洗文本和基础分段。

### 模块二：关键信息提取

必选字段：

- 姓名
- 电话
- 邮箱
- 地址

MVP 可选字段：

- 求职意向
- 期望薪资
- 工作年限
- 学历背景
- 项目经历

实现建议：

- 电话、邮箱优先用正则提取。
- 其他字段通过 AI 模型抽取。
- AI 返回必须被校验为结构化 JSON。

### 模块三：简历评分与匹配

- 接收岗位需求描述。
- 提取岗位关键词。
- 与简历文本和结构化信息做匹配。
- 返回总分、命中关键词、缺失关键词和简短摘要。

### 模块四：结果返回

- 所有结果以 JSON 返回。
- 错误信息结构化。
- MVP 不实现 Redis 缓存。

### 模块五：前端页面

- 上传 PDF。
- 展示解析状态。
- 展示简历信息。
- 输入岗位 JD。
- 展示匹配结果。

## 暂不实现

- 批量上传。
- Redis 缓存。
- 数据库。
- 登录和权限。
- OCR 扫描版 PDF 识别。
- 简历文件长期存储。
- 复杂可视化报表。
- 候选人管理后台。
- Vercel AI SDK 流式聊天。

## 验收标准

- 可以通过前端上传 PDF。
- 可以看到解析出的文本和结构化信息。
- 可以输入 JD 并得到匹配评分。
- `pnpm lint` 通过。
- `pnpm check-types` 通过。
- `pnpm build` 通过。
- 后端接口文档可通过 FastAPI OpenAPI 查看。

## 风险和处理

### 扫描版 PDF 无文本

MVP 不做 OCR。后端返回：

```txt
EMPTY_PDF_TEXT
```

前端提示用户上传可复制文本的 PDF 简历。

### AI JSON 不稳定

处理方式：

- Prompt 中明确要求只返回 JSON。
- 后端做 JSON 解析和字段兜底。
- 关键字段电话、邮箱使用正则兜底。

### Serverless 冷启动

处理方式：

- 控制依赖体积。
- 不在启动阶段加载大型模型。
- 模型调用走外部 API。
