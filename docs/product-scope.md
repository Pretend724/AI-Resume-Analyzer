# 产品范围

## 目标

AI Resume Analyzer 面向招聘筛选场景，提供从 PDF 简历到岗位匹配评分的完整分析流程：

1. 用户配置或测试 LLM 提供商。
2. 用户上传单个 PDF 简历。
3. 系统解析 PDF 文本。
4. 系统抽取候选人关键信息。
5. 用户输入岗位 JD。
6. 系统返回匹配评分、关键词分析和解释。

## 当前功能

### 简历上传与解析

- 支持上传单个 PDF 文件。
- 支持多页 PDF。
- 提取 PDF 文本。
- 清洗文本，包括空白字符、重复换行、不可见字符等。
- 支持中文和英文 section 标题识别。
- 返回原始文本、清洗文本和基础分段。

### 关键信息提取

基础字段：

- 姓名
- 电话
- 邮箱
- 地址

扩展字段：

- 求职意向
- 期望薪资
- 工作年限
- 学历背景
- 项目经历

实现方式：

- 电话、邮箱使用规则兜底。
- 其他字段优先通过 LLM 抽取。
- LLM 返回必须被校验为结构化 JSON。
- LLM 不可用时返回规则兜底结果和 warning。

### 简历评分与匹配

- 接收岗位需求描述。
- 提取岗位关键词。
- 与简历文本和结构化信息做匹配。
- 返回总分、命中关键词、缺失关键词和摘要。
- 使用 LLM 对规则评分进行增强。
- LLM 不可用时保留规则评分。

### LLM 提供商配置

- 支持从页面配置 Base URL、Model 和 API Key。
- 支持连接测试。
- 支持恢复后端默认配置。
- API Key 不在页面回显。
- 运行时配置保存在当前后端进程内。

### 前端工作台

- 三步式流程：上传、解析、评分。
- 展示解析状态。
- 展示简历信息。
- 展示清洗文本。
- 输入岗位 JD。
- 展示匹配结果。
- 展示结构化 JSON 数据。

## 当前不支持

- 批量上传。
- Redis 缓存。
- 数据库。
- 登录和权限。
- OCR 扫描版 PDF 识别。
- 简历文件长期存储。
- 复杂可视化报表。
- 候选人管理后台。
- 流式聊天交互。

## 风险和处理

### 扫描版 PDF 无文本

当前不做 OCR。后端返回：

```txt
EMPTY_PDF_TEXT
```

前端提示用户上传可复制文本的 PDF 简历。

### AI JSON 不稳定

处理方式：

- Prompt 中明确要求只返回 JSON。
- 后端做 JSON 解析和字段兜底。
- 关键字段电话、邮箱使用规则兜底。

### Serverless 冷启动

处理方式：

- 控制依赖体积。
- 不在启动阶段加载大型模型。
- 模型调用走外部 API。
